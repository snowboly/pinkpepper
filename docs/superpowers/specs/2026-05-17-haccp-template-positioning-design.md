# HACCP Template Positioning Design

## Goal

Reposition the existing `/features/haccp-plan-generator` page to target `HACCP plan template` intent more directly without changing the URL or removing the product-led software story.

## Context

Recent Google Ads keyword data showed stronger click-through behavior for `haccp plan template` than for `haccp plan generator`. The current page is heavily generator-led in its metadata, hero copy, headings, and long-form body. That positioning likely underuses the stronger template-oriented demand signal.

## Decision

Keep the existing feature URL and product category, but shift the page hierarchy:

- Primary search intent: `HACCP plan template`
- Secondary search intents: `create a HACCP plan`, `food safety plan template`, `HACCP template for small business`
- Product mechanism: PinkPepper as the software that helps tailor and generate a usable draft from that template structure

## Content Strategy

### Metadata

- Title should lead with `HACCP Plan Template`
- Description should describe PinkPepper as the way to turn that template into a tailored draft

### Hero

- H1 should lead with `HACCP Plan Template for Small Food Businesses`
- The supporting copy should frame PinkPepper as a faster way to tailor the template to products, process steps, hazards, and controls
- Support chips should include template-oriented language

### Body

The body should keep the existing substantive depth, but section framing should change:

1. `What This HACCP Plan Template Includes`
2. `How to Create a HACCP Plan from This Template`
3. `Why a Tailored Template Beats a Generic Download`
4. `Who This HACCP Template Is For`
5. `Built for UK and EU Compliance`
6. `Start with the Template, Then Refine the Draft`

## Constraints

- Keep the existing URL and page structure broadly intact
- Avoid making the feature page sound like a generic free PDF download
- Preserve the product differentiation around grounded guidance, tailored draft generation, and optional expert review
- Avoid creating direct cannibalization with `/resources/haccp-plan-template`

## Validation

Success for this change is:

- metadata and hero clearly align to `template` intent
- body headings and opening paragraphs reinforce that intent
- the page still reads as a product page, not a static-resource page

## 1. Purpose

To establish effective traceability systems and product recall procedures to enable rapid identification and withdrawal of unsafe or non-compliant food products, ensuring compliance with EC 178/2002 Article 18 and protecting consumer safety.

## 2. Scope

Applies to all food products manufactured, handled, or sold by the food business, covering ingredient traceability, batch coding, distribution records, and recall procedures.

## 3. Responsible Persons

| Role | Responsibility |
|---|---|
| Manager / Owner | Authorise recall decisions; lead recall team; liaise with authorities |
| Production Manager | Implement batch coding; maintain production records; conduct traceability tests |
| QA / Technical | Investigate incidents; assess risk; verify traceability data |
| Sales / Distribution | Notify customers; arrange product retrieval; provide distribution records |
| All staff | Record batch codes accurately; report traceability failures |

## 4. Traceability Principles

### One Step Back, One Step Forward

The food business must be able to identify:
- **One step back:** Where ingredients and raw materials came from (supplier name, batch/lot code, delivery date)
- **One step forward:** Where finished products were sold or distributed (customer name, batch code, delivery date, quantity)

### Traceability Speed

Traceability information must be retrievable within **4 hours** of a request.

### Batch Coding

All products manufactured or repacked must have a unique batch code or lot number enabling full traceability.

---

## 5. Supplier Traceability (One Step Back)

### Goods Receiving Records

For every delivery received, record:
- Supplier name and address
- Date received
- Product name and description
- Quantity received
- Supplier's batch code or lot number
- Best before / use by date
- Delivery note or invoice reference
- Temperature on arrival (for chilled/frozen goods)

**Record:** Goods inward log (paper or electronic).

**Retention:** Minimum 2 years (or product shelf life + 6 months, whichever is longer).

### Supplier Batch Codes

- All ingredient packaging must display supplier's batch or lot code
- If ingredients received in bulk (liquids, powders): record batch code on delivery documentation
- Ingredients stored with batch codes visible and identifiable
- FIFO (First In, First Out) rotation maintained; batch codes readable throughout storage

### Supplier Traceability Verification

- Conduct traceability exercise with each supplier annually
- Request traceability information for a random delivery
- Verify supplier can provide full traceability (one step back to their suppliers) within 4 hours
- Record outcome; address failures with corrective action

## 6. Internal Traceability (Process Tracking)

### Batch Code Creation

A unique batch code is assigned to each production run or batch of product.

**Batch Code Format Example:**
- `DDMMYY-PP-BB`
  - **DDMMYY:** Production date
  - **PP:** Product code
  - **BB:** Batch number (sequential on the day)

**Example:** `060325-CH-01` = 6 March 2025, Chicken product, Batch 01

### Production Records

For each batch produced, maintain a **batch production record** including:
- Batch code
- Product name
- Date and time of production
- Quantity produced
- Ingredients used (name, quantity, supplier batch codes)
- Packaging used (supplier, batch codes)
- Equipment used
- Staff on shift
- Quality checks conducted (temperature, weight, visual inspection)
- Any deviations or incidents
- Sign-off by supervisor

**Retention:** Minimum 2 years or product shelf life + 6 months.

### Linking Ingredients to Finished Product

Use production records or electronic system to link:
- Supplier ingredient batch codes → Internal batch code → Customer delivery

**Example Traceability Chain:**
1. Flour batch ABC123 from Supplier A received 1 March
2. Used in Bread batch 060325-BR-01 produced 6 March
3. Delivered to Customer X on 7 March (10 cases)

If Supplier A recalls flour batch ABC123, the food business can identify:
- Which finished product batches used that flour
- Which customers received those batches
- Quantity to recall

## 7. Batch Code Marking

### Product Labelling

- All finished products must display the batch code
- Batch code must be legible, indelible, and easily found (typically near date mark)
- Batch code printed using date coder, inkjet, or pre-printed labels

### Batch Code Verification

- Conduct daily checks to verify batch codes correct and legible
- Check date coder settings at start of each production run
- Record batch code checks on production record

---

## 8. Customer Traceability (One Step Forward)

### Sales and Distribution Records

For every sale or dispatch, record:
- Customer name and address (including contact phone number)
- Date of sale or dispatch
- Product name
- Batch code
- Quantity sold (number of units, cases, or weight)
- Invoice or delivery note number

**Retention:** Minimum 2 years or product shelf life + 6 months.

### Direct Sales (Retail or Online)

- For retail sales where customer details not collected: record date and batch code on till receipt or sales log
- Online sales: link order number to batch code dispatched
- Traceability forward limited to "sold to public on [date], batch code [XXX]" (full recall via public notification)

## 9. Traceability Testing

### Frequency

Conduct internal traceability tests **quarterly** (or more frequently for high-risk products).

### Procedure

1. **Select a random finished product batch** (batch code chosen at random)
2. **Trace forward:** Identify all customers who received that batch and quantities
3. **Trace backward:** Identify all ingredients (including supplier batch codes) used in that batch
4. **Time the exercise:** Record time taken to retrieve full traceability data
5. **Verify accuracy:** Check records against physical stock or delivery notes
6. **Record results:** Traceability test log (batch code tested, time taken, outcome, any failures identified)

### Target

Full traceability achieved within **4 hours**.

### Corrective Action

If traceability test fails (incomplete records, missing batch codes, time >4 hours):
- Investigate root cause (missing documentation, staff training gap, system failure)
- Implement corrective action (improve record-keeping, retrain staff, upgrade system)
- Retest within 1 month

## 10. Product Recall Procedure

### When to Recall

A product recall is initiated when:
- Product presents a serious health risk (allergen error, pathogen contamination, foreign body hazard)
- Product is mislabelled (incorrect allergen information, incorrect use-by date)
- Supplier issues a recall of an ingredient used in the product
- Enforcement authority requests recall
- Customer complaints indicate a safety issue affecting multiple batches

### Recall Decision

**Manager / Owner** makes the final recall decision in consultation with:
- QA / Technical (risk assessment)
- Enforcement authority (Environmental Health Officer or FSA)
- Legal or insurance advisor (if applicable)

**Urgency:** Decision made within **2 hours** of identifying the issue.

## 12. Recall Execution

### Step 1: Recall Initiation

1. **Stop production** of affected product
2. **Quarantine all stock** of affected batch(es) on site (label clearly: "QUARANTINED — DO NOT USE")
3. **Activate recall team** (emergency meeting within 2 hours)
4. **Notify enforcement authority** (Environmental Health Officer or FSA) immediately

### Step 2: Identify Affected Product

1. Identify affected batch code(s)
2. Use traceability records to determine:
   - Total quantity produced
   - Quantity remaining in stock (quarantine)
   - Quantity distributed (identify all customers)
3. Expand recall if necessary (if multiple batches affected or root cause unclear)

### Step 3: Risk Assessment

QA / Technical assesses:
- Nature of hazard (allergen, pathogen, foreign body, chemical)
- Severity (life-threatening, serious harm, minor risk)
- Vulnerable groups affected (children, elderly, allergic consumers)
- Likelihood of consumption (product shelf life, distribution time)

**Risk Categories:**
- **High Risk:** Serious or life-threatening hazard; wide distribution; immediate action required
- **Medium Risk:** Moderate hazard; targeted distribution; urgent action required
- **Low Risk:** Minor non-compliance; limited distribution; controlled withdrawal

### Step 4: Customer Notification

Draft recall notification including:
- Product name and description
- Batch code(s) affected
- Pack size and barcode (if applicable)
- Best before or use-by dates
- Reason for recall (clear explanation of hazard)
- Action required (stop sale, return to supplier, destroy)
- Contact details for queries

**Distribution:**
- **Phone calls** to all affected customers (immediate for high-risk recalls)
- **Written confirmation** (email or fax) within 2 hours
- **Follow-up:** Confirm customer received notification and has taken action

### Step 5: Public Notification (if applicable)

For recalls involving products sold directly to consumers:
- Draft **public recall notice** in consultation with enforcement authority
- Publish on Food Standards Agency recall database
- Issue press release (if high risk or wide distribution)
- Post notice on company website and social media
- Provide point-of-sale notices to retailers

### Step 6: Product Retrieval

- Customers return affected product or destroy on site (provide destruction instructions)
- Track returned product (quantity, batch codes, condition)
- Quarantine returned product; store separately from other stock
- Provide credits or refunds to customers

### Step 7: Disposal

- Affected product destroyed or rendered unfit for human consumption
- Methods: incineration, commercial waste disposal (Category 3 ABP if animal products), or denatured and sent to landfill
- Obtain disposal certificate
- Record batch codes and quantities destroyed

### Step 8: Effectiveness Check

- Within **7 days**, verify all customers notified and product retrieved or destroyed
- Contact non-responsive customers (follow-up phone calls, site visits if necessary)
- Calculate recall effectiveness rate:
  - **Effectiveness = (Quantity retrieved / Quantity distributed) × 100%**
  - **Target:** ≥95% for high-risk recalls

### Step 9: Root Cause Investigation

- Investigate how the incident occurred
- Identify failures in procedures, staff training, or supplier controls
- Implement corrective and preventive actions (CAPA)
- Review and update HACCP plan, SOPs, or specifications

### Step 10: Recall Report

Prepare final recall report including:
- Incident description and timeline
- Batches affected and quantities
- Customers notified
- Quantity retrieved and effectiveness rate
- Root cause and corrective actions
- Lessons learned

Submit report to enforcement authority; retain for **5 years**.

## 13. Product Withdrawal

A **product withdrawal** is a removal of product for quality (not safety) reasons:
- Off-specification (weight, colour, taste)
- Packaging error (minor label error not affecting safety)
- Commercial decision (product discontinuation)

**Procedure:** Similar to recall but lower urgency; enforcement authority notification may not be required (seek advice).

---

## 14. Mock Recall Exercise

### Frequency

Conduct **mock recall exercise annually** to test procedures and traceability.

### Procedure

1. Simulate a recall scenario (e.g. "Batch 010325-CH-02 found to contain undeclared milk allergen")
2. Activate recall team
3. Use traceability records to identify affected customers
4. Time the exercise
5. Draft recall notification (not sent)
6. Review effectiveness; identify gaps in procedures or records

### Target

Full traceability and draft notification within **4 hours**.

### Debrief

Review findings with recall team; implement improvements; update recall plan.

## 16. Training

All staff must receive training on:
- Importance of accurate record-keeping for traceability
- Batch code recording and verification
- Incident reporting

**Recall team members** receive additional training on:
- Recall procedures and team roles
- Mock recall participation
- Communication with authorities

**Frequency:** Induction + refresher annually; mock recall annually.
